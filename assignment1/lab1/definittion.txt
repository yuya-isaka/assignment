1.#include <iostream>

2.flowchart

演算データ、処理の流れ、装置などを表現するために記号を用いて表したもの。

3.syntax

プログラミング言語などの人工言語で定められている構文規則。（文法）

4.IDE
統合開発環境　プログラムを作るために必要なものが全部詰まった開発環境（コンパイラとかリンカとか）

5.main function
クラス外で定義されるグローバル関数　通常最初に実行される関数

6.comments
ソースコード内でメモを取った部分のこと。通常、//を使う。

7.expression
式　（ソースコード内での式）

8.operator
演算子

9.case sensitive
英文字の大文字と小文字を区別する。

10.pseudocode
擬似コード　コードを書かずに言葉だけで記述したもの

11.camelCase
アルファベットで複合語やフレーズを表記する際、各単語や要素語の先頭の文字を大文字で表記する手法のことである。

12.underscore_case
スネークケースのこと。単語間をアンダースコアでつないで表記する手法のこと。

13.cin.get()
キーボードから一文字だけ文字を読み込め　という意味。
ユーザの入力を待ち、ユーザが文字を入力したら、この関数の書かれている場所にその文字を置いてくれる

14.return 0
戻り値に値０を設定する。例えば、int main() 関数では、main関数がint型、つまり整数を持つ関数として定義されており、return 0とすることで、
main( ) の戻り値に int 型の値 0 を設定する、すなわち、main( ) 関数を呼び出した者に対して、「main( ) は 0（という int 型の値）である」事を伝える。ここでいう関数を呼び出したものは,
OS を指すことが多い。
OS において、あるプログラムが正常終了した事を意味する結果値は、0 が採用されている事が多く（例外はある）、return 0; とする事により、「main( ) は 0 だった」という事になり、プログラムが正常終了した、という事になる。

15.console window
ソースプログラムの実行画面

16.endl
改行

17.literal
プログラムのソースコードにおいて使用される、数値や文字列を直接に記述した定数のこと
変数の対義語で、基本的には変更されないことを前提としている。

18.variable
変数。リテラルの対義語。
プログラムのソースコード中で、データを一時的に記憶しておくための領域に固有の名前を付けたもの

19.text file
ある文字コードで文字として規定できる範囲のデータのみを含むファイルのこと。内容全体を文字として表示、編集でき、人間が容易に読み書きできる。
